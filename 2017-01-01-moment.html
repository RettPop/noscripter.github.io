新的一年，从研究moment模块开始

看node.js模块一定是从看源码的package.json文件开始，node.js在查找模块时会按照package.json文件main字段指定的文件来查找入口文件：

  "main": "./moment.js",

也就是说npm install moment后我们会引入moment.js这个文件。

这是一个4K+行的模块，不要怕，它的主要结构其实很简单：

(function (global, factory) {
  typeof exports === 'object' && typeof module !== 'undefined' ? module.exports = factory() :
    typeof define === 'function' && define.amd ? define(factory) :
    global.moment = factory()
}(this, (function () {
  /* moment 定义函数 */
})));

这种用法被称为immediately-invoked function expression，这种用法的好处在于避免变量提升污染全局环境，但是又能将方法暴露出去公开访问。

  typeof exports === 'object' && typeof module !== 'undefined' ? module.exports = factory() :
    typeof define === 'function' && define.amd ? define(factory) :
    global.moment = factory()

这是对运行环境（node.js还是amd）的判断。

https://addyosmani.com/writing-modular-js/这篇文章较好的介绍了JavaScript模块化编程的几种形式，现在看来ES Harmony Module和Class已经成为了标准。 2016年，React、Node.js的普及推进了ES 2016 Module和Class的使用和标准化。感谢曾经『百家争鸣』的其他模块形式，前端模块化编程的统一和标准化将进一步提升开发者体验、促进社区的良好发展。

接着看moment定义函数的实现，主要结构简化如下：

  'use strict';
  var hookCallback;
  function hooks() {
    return hookCallback.apply(null, arguments);
  }
  /* 具体的工具方法和实现逻辑 */
  return hooks;

这里的逻辑比较复杂，通过多个函数：setHookCallback->createLocal->createLocalOrUTC->createFromConfig->：

  var res = new Moment(checkOverflow(prepareConfig(config)));
  if (res._nextDay) {
    // Adding is smart enough around DST
    res.add(1, 'd');
    res._nextDay = undefined;
  }
  return res;

其中Moment构造函数为：

function Moment(config) {
  copyConfig(this, config);
  this._d = new Date(config._d != null ? config._d.getTime() : NaN);
  if (!this.isValid()) {
    this._d = new Date(NaN);
  }
  // Prevent infinite loop in case updateOffset creates new moment
  // objects.
  if (updateInProgress === false) {
    updateInProgress = true;
    hooks.updateOffset(this);
    updateInProgress = false;
  }
}

从这里可以看出，底层还是依赖于JavaScript的Date API来实现的。其他工具函数和具体转换的逻辑，感兴趣的可以自己去扒代码研究了。
